# Meteor Security CLI - By Spark Systems (برق للأنظمة) 🚀⚡


## 1. نظرة عامة (Overview)
أداة Meteor هي إطار عمل أمني (Security Framework) وواجهة سطر أوامر (CLI) صُممت لتمكين مديري الأنظمة في **Spark Systems** من اكتشاف الثغرات الأمنية، فحص المنافذ المفتوحة، ربط العمليات بالمسارات التنفيذية، وتحليل السجلات الخاصة بالنظام لاكتشاف الأنشطة المشبوهة.

## 2. الهيكلية المعمارية (Architecture & SOLID Principles)
تم تصميم النظام ليكون مقسماً إلى طبقات (Layers) مفصولة عن بعضها لتسهيل عملية الصيانة والتطوير، وهذا يتماشى مع مفاهيم هندسة البرمجيات الحديثة:

### أ. مبدأ المسؤولية الفردية (Single Responsibility Principle)
تم تقسيم الوظائف الأساسية إلى كائنات (Classes) مستقلة، حيث يؤدي كل كائن وظيفة واحدة فقط:
- **`ScannerEngine`**: مسؤول عن فحص المنافذ المفتوحة محلياً.
- **`DeepScanner`**: مخصص للفحوصات العميقة المعتمدة على الـ SYN packets وتدقيق تكامل العمليات (Root Required).
- **`ProcessManager`**: وظيفته تنحصر في جلب معلومات العمليات (PIDs) ومساراتها التنفيذية.
- **`LogAnalyzer`**: مخصص لقراءة سجلات النظام والبحث التلقائي عن أنماط مشبوهة.
- **`HardwareAudit`**: يتعامل مباشرة مع ملفات النظام الخاصة بالمعالج لكشف ثغرات الهاردوير.
- **`EncryptedVault`**: يدير المفاتيح السرية بتشفير عسكري AES-256.
- **`PasswordAnalyzer`**: محرك رياضي لتقييم قوة كلمات المرور وتقدير زمن كسرها.
- **`IdentityGuard`**: محرك بحث عن التسريبات يستخدم تقنيات K-Anonymity.
- **`KillChainAnalyzer`**: محرك الارتباط (Correlation Engine) الذي يجمع كل الإشارات لتقييم الخطر العام.

### ب. مبدأ الانفتاح/الانغلاق (Open/Closed) والمقلوب الاعتمادي (Dependency Inversion)
تعتمد المحركات الأساسية على واجهات مجردة (Abstract Interfaces).
- **`BaseProvider`**: يحدد الدوال الأساسية.
- **`PrivilegeProvider`**: يتعرف على مستوى امتيازات المستخدم لتفعيل أو تعطيل "الوضع القتالي".
- بفضل هذا التصميم، يمكن إضافة أي ميزة أو دعم أي نظام جديد بمجرد إنشاء Class جديد دون المساس بالكود القديم.

## 3. شرح الوحدات المتقدمة (Advanced Modules)

### 3.1 الخزنة المشفرة (Encrypted Vault)
- **المكان**: `core/vault.py`
- **التقنية**: تستخدم `PBKDF2HMAC` لاشتقاق المفتاح من الباسوورد مع ميزة الـ `Salt` لمنع هجمات المعاجم المسبقة. يتم تشفير البيانات باستخدام `Fernet` (AES-256 in CBC mode مع HMAC).
- **الوظيفة**: حفظ مفاتيح Shodan و HIBP بشكل لا يمكن قراءته حتى لو تم الوصول للملف الفيزيائي.

### 3.2 فحص الهاردوير (Hardware Audit)
- **المكان**: `providers/hardware.py`
- **التقنية**: قراءة مباشرة من `/sys/devices/system/cpu/vulnerabilities/`.
- **الوظيفة**: الكشف عن حالة النظام تجاه هجمات Spectre, Meltdown, L1TF, وغيرها، وتوضيح حالة التخفيف (Mitigation).

### 3.3 محرك فحص كلمات المرور (Password Engine)
- **المكان**: `core/password.py`
- **التقنية**:
    - **Entropy Calculation**: حساب رياضي لعدد المحاولات الممكنة.
    - **Crack Time Estimation**: محاكاة لسرعة معالجات GPU الحديثة.
    - **HIBP K-Anonymity**: إرسال أول 5 رموز من الهاش فقط لضمان عدم خروج كلمة المرور من جهازك أبداً.

### 3.4 الوضع القتالي (Deep Scanning / Combat Mode)
- **المكان**: `core/scanner.py` & `cli.py`
- **المميزات**:
    - **SYN Scanning**: استخدام مكتبة `scapy` لبناء حزم شبكية مخصصة.
    - **Process Integrity (Advanced)**: الكشف عن هجمات **Process Hollowing** عبر تحليل خرائط الذاكرة (`/proc/pid/maps`) وربطها بالملفات التنفيذية لاكتشاف الأكواد المحقونة في الذاكرة المجهولة (Anonymous executable regions).

### 3.5 محرك الدرع (The Shield - Hardening Engine)
- **المكان**: `core/shield.py`
- **الوظيفة**: محرك تحليل لثغرات النظام التشغيلية (OS Hardening).
- **الفحوصات**: التحقق من إعدادات SSH، حالة ASLR، حماية IP Spoofing، وقبول تحويلات ICMP. يعطي تقييماً رقمياً (Score) واقتراحات فورية للإصلاح.

### 3.6 استخبارات التهديدات الخارجية (External Threat Intel)
- **المكان**: `external/`
- **المنصات المدعومة**:
    - **VirusTotal**: فحص الملفات والروابط.
    - **AbuseIPDB**: التحقق من سمعة عناوين الـ IP.
    - **AlienVault OTX**: جلب مؤشرات الاختراق من المجتمع الأمني.
- **التكامل**: يتم ربط هذه البيانات بـ `KillChainAnalyzer` لرفع درجة الخطر تلقائياً في حال اكتشاف تطابق مع تهديد معروف عالمياً.

## 4. واجهة سطر الأوامر (CLI - Nebula Dashboard)
تم إدراج أوامر جديدة للتعامل مع الاستخبارات الخارجية:
- **`meteor threat <indicator>`**: لفحص أي IP أو Hash أو URL عبر جميع المنصات المدعومة في آن واحد.
- **`meteor vault add <service>`**: لدعم إضافة مفاتيح VT و AbuseIPDB و OTX بشكل مشفر.
